Fix the RAG indexer duplicate-ID bug that caused live CIQ transcript indexing to fall back to mock mode.

## Problem
When running `python main.py --index-transcripts --live`, the live CIQ transcripts from wrds_client.query_all_transcripts() produce duplicate chunk IDs in ChromaDB, causing the indexing to fail and fall back to mock.

## Root cause investigation
1. Read rag/indexer.py and find how chunk IDs are generated (the _iter_chunks function)
2. Read data/wrds_client.py query_all_transcripts() to see what format the live data returns
3. The likely issue: live transcripts may have quarter strings or dates in a different format than mock data, causing ID collisions when multiple transcript components map to the same quarter string, OR the same transcriptid appears multiple times.

## Fix
- Make chunk IDs globally unique by incorporating enough distinguishing info (e.g. transcriptid, or a hash of the text content, or a counter)
- Ensure query_all_transcripts() deduplicates transcripts if the SQL returns duplicates
- After fixing, verify by running: python main.py --index-transcripts --live
- Then run: python main.py --evaluate --live
- Update README.md to remove the note about RAG falling back to mock mode
