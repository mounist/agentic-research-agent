## What to build

Create a new file `agent/loop_langgraph.py` that reimplements the same agent behavior using LangGraph's StateGraph. The existing agent/loop.py uses a manual while-loop with Claude tool_use API calls. The new version should express the same logic as a graph.

## Architecture

Study the existing agent/loop.py carefully first. Then build:

1. **State schema**: Define an AgentState TypedDict that holds:
   - messages (conversation history)
   - tool_sequence (list of tool names called)
   - iteration_count (int)
   - input_tokens (int)
   - output_tokens (int)  
   - final_report (str or None)
   - data_mode (str)
   - query (str)
   Do NOT put EvalRecord in state. Store raw tracking fields, and assemble EvalRecord at the end in run_agent_langgraph().

2. **Graph nodes**:
   - `reason_node`: Claude API call with tool_use. This handles ALL iterations including the first one (the agent naturally outputs a research plan as text in its first response alongside tool_use blocks — do NOT create a separate plan node, as that would add an extra API call that doesn't exist in the original). Must replicate the tool_budget_nudge logic: when len(unique tools in tool_sequence) >= 7, inject the TOOL_BUDGET_NUDGE message. Must replicate the force-report logic: when iteration_count >= MAX_ITERATIONS, inject FORCE_REPORT_PROMPT.
   - `tool_node`: Dispatches tool calls using the existing dispatch() function from agent/loop.py. Do NOT rewrite tool dispatch logic.
   - (No separate plan_node or synthesize_node needed — reason_node handles these cases via conditional message injection, same as the original loop.)

3. **Conditional edges**:
   - START → reason_node
   - reason_node → tool_node (if response contains tool_use blocks)
   - reason_node → END (if response is end_turn with no tool_use → final report)
   - tool_node → reason_node (loop back for next iteration)

4. **Entry point**: `run_agent_langgraph(query: str, data_mode: str = "live") -> tuple[str, EvalRecord]` — SAME return signature as existing `run_agent()`.
